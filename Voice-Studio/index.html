How it works
  1. UI calls GET /voices to list available voice WAVs.
  2. Click Generate → POST /generate returns a job_id.
  3. UI polls GET /status/{job_id} until done.
  4. Audio loads from /files/{job_id}.wav.
Recommended for large teams
  • Keep a queue (single synthesis at a time on CPU).
  • Set max text length (e.g. 800–1500 chars/request).
  • Use IIS auth / SSO, or an API key header.
  • Add MP3 export later to reduce file sizes.
If your UI is hosted on IIS and API is on same domain/path, CORS is not an issue. If cross-domain, enable CORS in FastAPI.
